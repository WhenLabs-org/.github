# WhenLabs

Six developer tools. One install.

```bash
npx @whenlabs/when install
```

## The Toolkit

| Tool | What it does |
|------|-------------|
| [velocity](https://github.com/WhenLabs-org/velocity) | MCP server — gives coding agents self-awareness of their execution speed |
| [stale](https://github.com/WhenLabs-org/stale) | Detect documentation drift in your codebase |
| [envalid](https://github.com/WhenLabs-org/envalid) | Type safety for .env files |
| [berth](https://github.com/WhenLabs-org/berth) | Port conflict detection and resolution |
| [aware](https://github.com/WhenLabs-org/aware) | Auto-detect your stack, generate AI context files |
| [vow](https://github.com/WhenLabs-org/vow) | Scan dependency licenses against plain-English policies |

All six ship together as **[when](https://github.com/WhenLabs-org/when)** — a single CLI that wraps them into one unified workflow.

## Philosophy

Small, focused tools that do one thing well. Each works standalone. Together they cover the gaps that slow developers down — stale docs, env bugs, port conflicts, license surprises, missing context files, and no visibility into agent performance.

Everything is open source under MIT.
